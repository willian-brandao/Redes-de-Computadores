## Introdução

Arquitetos e Administradores de redes devem ser capazes de mostrar como suas redes irão ser. Eles precisam ser capazes de ver facilmente quais componentes conectam com outros componentes, onde ele irão ser localizados e como eles irão se conetar. Diagramas de Redes geralmente usam símbolos para representar os diferentes dispositivos e conexões que formam a rede. 

<img width="823" height="588" alt="image" src="https://github.com/user-attachments/assets/f67d0832-e210-4e20-9373-ed9689d154f2" />

Um diagrama provês um caminho fácil para entender como dispositivos se conectam em uma rede grande. Esse tipo de imagem é conhecida como diagrama de topologia. A habilidade de reconhecer a representação lógica de componentes físicos de rede é fundamental para ser capaz de visualizar a organização e operação da rede. 

Em conjunto com essas representações, terminologia especializada é usada para descrever como cada dispositivo e mídia conectam entre si. 

* Placa de Rede (NIC) - NIC conecta fisicamente o dispositivo final à rede.
* Porta Física - Um conector no dispositivo de rede onde a mídia se conecta em um dispositivo final ou a outro dispositivo de rede.
* Interface - Porta especializada no dispositivo de rede que conecta em uma rede. Como os roteadores conectam redes. as portas conectam no roteador são referidas como interfaces de redes.

## Diagrama de Topologia

Diagramas de Topologia são documentação obrigatória para qualquer um que trabalhe com redes. Eles provêm um mapa visual de como a rede é conectada. Existem dois tipos de diagramas de topologia: física e lógica.
