## Comunicação e Redes

### Conceito de Redes de Comunicação 

**Comunicação** é quando a informação é transferida e trocada entre pessoas, entre pessoas e coisas, e entre coisas através de um determinado meio e comportamento. 

**Comunicação de Redes** é a comunicação entre dispositivos terminais através das redes de computadores. Por exemplo:

### Processo de Transferência de Informação 

O processo de transferência de dados é similar ao processo de transferência de objetos no mundo real. Assim como no mundo real, um objeto que é transferido entre um local e outro, é embalado, entregue a um centro de distribuição e recebe etiquetas para identifica-lo para que ele chegue ao destino final de forma correta e segura. Após todo esse processo, ele é desembalado, e o conteúdo é entregue ao destinatário. 

Quando um endpoint precisa transmitir dados ou arquivos, esses dados são encapsulados,  no mundo real seria como se eles fossem embalados, depois recebem um header ou tails para formar o pacote, depois esse pacote é transferido para um roteador de gateway que conecta o endpoint à internet, depois de passar pela internet esse pacote chega a outro roteador ligado ao destinatário, assim sendo desencapsulado,  tendo os seus identificadores (tail e header) removidos, assim entregando o conteúdo ao destinatário. 

#### Termos Comuns

* Payload - Informação Transmitida.

* Pacote - Unidade de Dados trocada e transmitida na rede. 

* Header - Segmento de informação adicionada antes do payload de dados.

* Tail - Segmento de informação adicionada depois de payload de dados. 

* Encapsulamento - Processo de adicionar header e tail ao payload de dados para formar um novo pacote. 

* Desencapsulamento - Processo de remover o header e o tail do pacote para obter o payload de dados. 

* Gateway - Dispositivo de Rede que provê funções como conversão de protocolos, seleção de rotas e transferência de dados. 

* Endpoint - Dispositivo final de sistema de comunicação de dados, usado como origem e destino dos dados. 


### Conceito de Rede de Comunicação de Dados

Uma rede de comunicação de dados possui diversos dispositivos: computadores, roteadores, switches, firewalls, controladores de acesso(ACs), access points (APs), impressoras e servidores.

Esses dispositivos ficam interconectados entre si para estabelecer comunicação. Os computadores são conectados por meio de switches de acesso, switches gerais e switches de agregação, e depois conectados à internet. Na internet existem diversos roteadores e firewalls. Esses dispositivos formam essa rede de comunicação de dados. 

No caso da transmissão sem fio, controladores e pontos de acesso também estão envolvidos. A função da rede de comunicação de dados é implementar comunicação de dados.

* Switches - O dispositivos que conecta os usuários finais à rede. É usado para acessar a rede e troca de frames de dados. O switch é um dispositivo de camada 2. 
  
   <img width="489" height="368" alt="image" src="https://github.com/user-attachments/assets/5ae9eeaf-1575-42b4-b40f-328482d71cd9" />

* Roteadores - Um dispositivo da camada de rede que encaminha pacote de dados na internet. Baseado nos endereços de destino em pacotes recebidos, um roteador seleciona um caminho para enviar o pacote para o próximo roteador ou destino. O último roteador no caminho é responsável por enviar o pacote para o host de destino.

   <img width="293" height="198" alt="image" src="https://github.com/user-attachments/assets/6a9e556b-6681-4e5b-a82f-ef316d44e5f2" />

  * Implementa comunicação entre redes do mesmo tipo ou de diferentes tipos.
  * Isolamento de domínios de broadcast.
  * Manutenção de tabelas de roteamento e executam protocolos de roteamento.
  * Selecionam rotas e encaminham pacotes de IP.
  * Implementando acesso WAN e tradução de endereço de rede.
  * Conectando redes de camada 2 estabelecidas através de switches.

* Firewall - Um dispositivo de segurança de rede usado para assegurar comunicação segura entre duas redes. Ele monitora, restringe e modifica fluxo de dados passando por eles para proteger informações, estruturas, e funcionamento de redes internas a partir de redes públicas.

## Tipos de Redes e Tipos de Topologia 

### Tipos de Redes 

#### LAN, MAN e WAN

Baseado em cobertura geográfica, redes podem ser classificadas dentro de redes de áreas locais (LAN), redes de áreas metropolitanas (MAN) e redes de áreas extensas (WAN).

|                LAN                |                MAN                |                WAN                |         
|-----------------------------------|-----------------------------------|-----------------------------------|
| Uma LAN é uma rede que consiste em computadores, servidores e dispositivos de redes em uma área geográfica. A cobertura de uma LAN é geralmente dentro de alguns metros quadrados. | Uma MAN é uma rede de comunicação de computadores estabelecida dentro de uma cidade. | Uma WAN geralmente cobre uma área geográfica extensa variando desde 10 kilometros quadrados até 1000 kilometros quadrados. Ela pode conectar redes de múltiplas cidades ou até redes de países. |
| Uma LAN tipíca inclui redes de escritórios de empresas, uma rede de uma cafeteria ou uma rede de uma casa | Uma MAN tipíca incluem MANs de banda larga, MANs educacionais, e linhas privadas governamentais de munícipios e províncias | A internet é uma típica WAN |

## Tipos de Topologia 

Estrutura de uma rede e a maneira como os dispositivos são conectados uns os outros.

<img width="237" height="189" alt="image" src="https://github.com/user-attachments/assets/bbed4c08-ebfe-4c9c-a924-3a9a1b53509e" />

Topologias de Redes são classificadas em estrela, anel, barramento, árvore, full-mesh, partial-mesh

<img width="556" height="218" alt="image" src="https://github.com/user-attachments/assets/f780e67c-5906-4a7c-bc45-b216d1326110" />

* Topologia estrela - Topologia de rede onde todos os dispositivos estão conectados diretamente a um nó central e todos os outros nós estão diretamente conectados. 

* Topologia de Barramento - Nesta topologia não existe um nó central, porém existe um barramento, onde todos os disposisitivos finais são conectados a esse barramento. Há uma desvantagem, quando dois dispositivos se comunicam entre si, os outros não podem se comunicar simultaneamente utilizando esse barramento.

* Topologia de anel - Todos os nós conectados entre si para formar um anel. Em uma rede de anel, a confiabilidade não é muito boa, se um link falhar, precisam ser utilizadas muitos hubs para comunicação. Se dois links falharem, a rede não estará mais conectada. 

* Topologia em árvore - Na topologia em árvore, um nó raiz, alguns ramos e nós folhas. Uma topologia e árvore é muito estruturada, mas a confiabilidade não é alta. Se o link for desconectado, esse nó não poderá mais transmitir nenhuma informação para esses nós.

* Rede Mesh Total - É a topologia mais robusta e confiável. Nessa topologia, na qual cada dois nós são conectados entre si, se um link falhar, o nó ainda pode ser comunicar com a outra extremidade através de outros links. Em uma rede mesh, a robustez é muito boa, mas o custo é alto devido ao excesso de links.

* Rede Mesh Parcial - Nessa topologia, alguns nós importantes têm links entre si, enquanto outros nós não têm links entre si.

No mundo real, uma rede pode ser uma topologia de rede combinada. Pode ser uma topologia em árvore e ter alguns links mesh entre os nós 

## Engenharia de Rede e Engenheiros de Rede

### Engenharia de Redes

Engenharia de Redes refere-se ao planejamento e ao projeto de soluções viávei com base nos requisitos das aplicaões de rede e nos padrões, especificações e tecnologias dos sistemas de rede de computadores, sob a orientação de métodos de engenharia de sistemas de informação e organizações completas, bem como à integração de dispositivos de hardware, software e tecnologias, para formar um sistema de rede com boa relação custo-benefício que atenda aos requisitos dos usuários.
